### Суть лабораторной работы
#### Цель: создать простую end-to-end систему, которая генерирует данные, сохраняет их и позволяет анализировать через Redash и Jupyter Notebook.

#### Компоненты системы:
1) **Генератор данных** (скрипт на Python'е) - генерирует данные и добавляет из базу данных (PostgresSQL)
2) **База данных** - хранит сгенерированные данные
3) **Redash** - dashboard'ы и визуализация

### Как запустить приложение?
На вашем устройстве должен быть установлен Docker Desktop.

Для работы приложения необходимо склонировать данный репозиторий 
(```git clone https://github.com/vstaff/data_analysis_lab.git```).

Установите необходимые зависимости для работы Python-скрипта (см. ```requirements.txt```) с помощью команды
```pip install requirements.txt```.

Далее нужно определить переменные окружения - данные из них подтягиваются Docker'ом для развертывания бд и подключения 
к ней. Создайте файл ```.env``` в корне проекта и добавьте необходимые переменные.

Пример того как должны выглядеть переменные окружения можете посмотреть в корне проект в файле ```.env.example```. 
Можно просто скопировать данные и вставить их в ```.env``` 

Далее необходимо запустить docker-контейнер:
```docker-compose up --build```.

После этого, если все было сделано корректно вы увидите в консоли логи, которые говорят о том что данные успешно 
добавляются в базу данных. В терминале должно быть что-то вроде:

```
data_generator_container  | Добавлен заказ: Плед (Дом) -> Южно-Сахалинск
data_generator_container  | Добавлен заказ: Плед (Дом) -> Артём
... и так далее ...
```

При каждом запуске контейнера новые данные добавляются к уже существующем в бд. Если вы хотите очистить бд - 
воспользуйтесь командой ```docker-compose down -v```.

Пока что функционал для визуализации и dashboard'ов не сделан *(компонент системы 3)*.
